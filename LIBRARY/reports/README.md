# Library Reports

This directory contains auto-generated reports about the repository activity and library statistics.

## Report Types

### Daily Reports
- **File**: `daily-YYYY-MM-DD.md`
- **Generated**: Daily at 3:00 AM UTC
- **Content**:
  - Files added/modified/deleted in the last 24 hours
  - Commit statistics
  - Activity summary

### Weekly Reports
- **File**: `weekly-YYYY-WNN.md` (where WNN is the week number)
- **Generated**: Every Friday at 3:00 PM UTC
- **Content**:
  - Repository activity for the week
  - Most modified files
  - Top contributors
  - Active areas of development

### Monthly Reports
- **File**: `monthly-YYYY-MM.md`
- **Generated**: First day of each month at 9:00 AM UTC
- **Content**:
  - Growth analysis for the month
  - Trends and week-by-week breakdown
  - Quality metrics
  - Recommendations for maintenance

## Report Generation

Reports are automatically generated by the [Library Report workflow](../../.github/workflows/library-report.yml).

You can also trigger report generation manually:
1. Go to GitHub Actions
2. Select "Library Report" workflow
3. Click "Run workflow"

## Note

Reports are automatically generated and should not be manually edited. They provide historical insights into repository development patterns.

*Reports are auto-generated and excluded from version control to reduce repository size*
