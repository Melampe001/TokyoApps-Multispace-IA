## Notes on Changelog Maintenance

This changelog documents all notable changes to the Tokyo-IA project. When making updates:

1. **Add entries under [Unreleased]** for changes that haven't been released yet
2. **Use semantic versioning** (MAJOR.MINOR.PATCH) for releases
3. **Categories**: Added, Changed, Deprecated, Removed, Fixed, Security
4. **Include dates** in YYYY-MM-DD format
5. **Reference issues/PRs** when applicable
6. **Move [Unreleased] items** to a new version section when releasing

### Category Definitions

- **Added**: New features or functionality
- **Changed**: Changes to existing functionality
- **Deprecated**: Soon-to-be removed features
- **Removed**: Removed features or functionality
- **Fixed**: Bug fixes
- **Security**: Security vulnerability fixes or improvements
